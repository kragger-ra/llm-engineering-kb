# llm-engineering-kb

База знаний по прикладному LLM-инжинирингу в формате DSWoK. Цель — корпус
учебного и справочного уровня («свободное владение») по трём слоям:
продукт (AI Product Engineer), модель (ML Engineer), инфраструктура.
Корпус пригоден для чтения человеком и для индексации в RAG учебного агента.

## Что здесь лежит

- `BUILD_SPEC.md` — задание для Claude Code: как итеративно собрать полный
  корпус нужной глубины. Это основной файл. Открой его в Claude Code и скажи
  «выполняй BUILD_SPEC.md».
- `TEMPLATE.md` — шаблон одной заметки (frontmatter + обязательные секции +
  планка глубины).
- `sources.md` — выверенный список источников по каждой теме (сиды для research).
- `sample/inference-serving.md` — **эталонный раздел на полную глубину**. Это
  планка качества. Каждая заметка в корпусе должна быть такой же плотной.
- `content/` — сюда Claude Code пишет готовые заметки (создаётся при сборке).

## Почему не один большой файл

Один файл = плохой RAG (конкурирующие чанки, потеря границ тем) и неудобное
чтение. Здесь — одна тема = один файл с frontmatter-метаданными, секции режутся
по заголовкам в самодостаточные чанки.

## Связь с исходным DSWoK

Базовый DSWoK (github.com/Erlemar/dswok) покрывает attention, negative sampling,
LoRA, RAG-пайплайн, Transformer, BERT, тематическое моделирование. Этот корпус
их **не дублирует** — ссылается «см. DSWoK §X.Y» и добавляет только прикладную
дельту. Правило анти-дублирования — в `BUILD_SPEC.md`.

## Как учиться по корпусу

1. Читать заметку целиком (суть → механика → практика → отказы).
2. По блоку «Вопросы для самопроверки» проверять, что владеешь, а не узнаёшь.
3. Прогонять код из заметки руками.
4. Идти по `prereqs` из frontmatter, если тема опирается на другую.
