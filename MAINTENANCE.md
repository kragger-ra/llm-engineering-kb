# MAINTENANCE — регламент поддержки корпуса

Корпус содержит факты, которые устаревают с разной скоростью. Чтобы числа и
рекомендации оставались верными, действует правило перепроверки.

## Правило

- **Волатильные заметки** (`volatile: true` во frontmatter) — перепроверять **раз в
  квартал** (≈90 дней). Это темы, где быстро меняются цены/железо, версии библиотек,
  фичи провайдеров, SOTA-методы и бенчмарки.
- **Остальные заметки** (`volatile: false`) — перепроверять **раз в полгода**
  (≈180 дней). Механика и математика устаревают медленно, но ссылки и контекст —
  тоже стоит сверять.

При перепроверке: обновить факты по ≥2 независимым источникам (как при сборке, см.
`BUILD_SPEC.md` §2–§3), поставить новую дату в `last_reviewed`, отметить в тексте,
если конкретное число привязано к дате.

## Как найти, что пора проверять

```bash
# Волатильные заметки и их даты последней проверки:
grep -rl 'volatile: true' content/ | while read f; do
  echo "$(grep -m1 last_reviewed "$f" | tr -d ' ')  $f"
done | sort
```
